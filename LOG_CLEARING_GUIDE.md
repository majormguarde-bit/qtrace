# Функционал очистки журнала администратора

## Описание
Добавлены две функции для очистки журнала администратора:
1. **Очистка по фильтрам** - удаляет записи, соответствующие текущим фильтрам
2. **Полная очистка** - удаляет весь журнал

## Использование

### Очистка по фильтрам
1. Перейдите в "Администрирование" → "Журнал"
2. Установите нужные фильтры (администратор, действие, дата и т.д.)
3. Нажмите кнопку "Очистить по фильтрам" (желтая кнопка)
4. Подтвердите удаление в модальном окне

**Пример**: Удалить все записи о сброшенных паролях за последний месяц
- Фильтр по действию: "Сброшен"
- Фильтр по дате: от 01.01.2026 до 30.01.2026
- Нажать "Очистить по фильтрам"

### Полная очистка
1. Перейдите в "Администрирование" → "Журнал"
2. Нажмите кнопку "Очистить всё" (красная кнопка)
3. Подтвердите удаление в модальном окне

⚠️ **ВНИМАНИЕ**: Это действие удалит ВСЕ записи журнала и не может быть отменено!

## Функциональность

### Кнопки в журнале
- **Применить фильтры** (синяя) - применить текущие фильтры
- **Сбросить** (серая) - очистить все фильтры
- **Очистить по фильтрам** (желтая) - удалить записи по фильтрам
- **Очистить всё** (красная) - удалить весь журнал

### Модальные окна подтверждения
Перед удалением показывается модальное окно с предупреждением:
- Для фильтрованной очистки: показывает количество записей для удаления
- Для полной очистки: показывает общее количество записей и предупреждение

### Логирование
Все операции очистки логируются:
- В файл `logs/admin_password_log.log`
- С информацией об администраторе, количестве удаленных записей и времени

## Примеры логов

```
[2026-01-30T23:30:45.123Z] Admin: admin | Action: logs_cleared_filtered | Records deleted: 15
[2026-01-30T23:31:20.456Z] Admin: admin | Action: logs_cleared_all | Records deleted: 127
```

## Права доступа
Только администраторы (ADMIN) и суперпользователи могут очищать журнал.

Попытка очистки без прав приведет к ошибке:
```
У вас нет прав для очистки журнала.
```

## Безопасность
- Все операции требуют подтверждения в модальном окне
- Все операции логируются с указанием администратора
- Удаленные данные невозможно восстановить
- Рекомендуется регулярно архивировать журнал перед очисткой
